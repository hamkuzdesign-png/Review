# Figma Design vs Implementation

Плагин для Figma, сравнивающий выбранный фрейм пиксель-в-пиксель (и, в будущих итерациях, семантически) с реальной реализацией: веб-страницей или экраном из APK.

Статус: реализован **режим "Веб"** — пиксельный diff + семантическое сопоставление слоёв Figma с DOM-элементами страницы (позиция, размер, цвет, шрифт, радиус скругления, отступы). Режим "APK" — в разработке (см. план в `.claude/plans`).

## Структура

- `packages/shared` — общие типы (контракт между плагином и бэкендом).
- `packages/server` — локальный Node-бэкенд:
  - `web/screenshot.ts` — живой скриншот страницы + DOM-дерево (Playwright), в одном проходе;
  - `web/domExtract.ts` — извлечение bbox/цветов/шрифтов/отступов из computed style каждого DOM-элемента;
  - `diff/` — пиксельный diff (`pixelDiff.ts`) и кластеризация расхождений в читаемые области (`clustering.ts`);
  - `match/matchEngine.ts` — сопоставление слоя Figma с DOM-узлом: точное совпадение имени/id/data-testid → совпадение текста → (только для изображений) сопоставление по пересечению координат между `hasImageFill` и `hasImage`;
  - `match/compareProps.ts` — сравнение размера/цвета/шрифта/радиуса/отступов/текста для каждой пары и генерация расшифровки на русском с привязкой к категории и разметкой (точка/линия-размер/дуги скругления);
  - `match/imageContent.ts` — пиксельное сравнение содержимого именно внутри bbox элемента (это ловит "не тот логотип/картинка", а не только цвет/размер);
  - `match/visualGate.ts` — если метаданные отличаются, но пиксели в этой области визуально одинаковы, расхождение не показывается (защита от шума несовпадающих метаданных);
  - `report/buildReport.ts` — сборка итоговых маркеров (семантика + непокрытые пиксельные области + "отсутствует"/"лишний элемент"), нумерация, обрезка кропов.
- `packages/plugin` — сам плагин Figma (main-поток + UI на React), плюс `reportRender.ts` — сборка отчёта из настоящих Figma-нод на холсте.

## Установка

```bash
npm install
npx playwright install chromium   # один раз, для живого скриншота веб-страниц
```

## Запуск

1. Запустите бэкенд:
   ```bash
   npm run dev:server
   ```
   Слушает `http://localhost:4517`. Проверить: `curl http://localhost:4517/health`.

2. Соберите плагин:
   ```bash
   npm run build --workspace=packages/plugin
   # или в режиме наблюдения:
   npm run dev:plugin
   ```

3. В Figma (desktop app): Plugins → Development → Import plugin from manifest… → выберите `packages/plugin/manifest.json`.

4. Выделите фрейм в Figma, откройте плагин, вкладка "Веб": укажите URL дев-сервера (или включите загрузку скриншота вручную, если URL недоступен для бэкенда), нажмите "Взять выделенный фрейм из Figma", затем "Сравнить".

5. После сравнения плагин автоматически создаёт на холсте рядом с выбранным фреймом новый фрейм-отчёт: обе картинки (Figma и реализация) с пронумерованными рамками поверх расхождений, плюс список расшифровок под ними. Это полноценные Figma-ноды — можно зумировать, комментировать, экспортировать как обычный фрейм. Мини-панель самого плагина показывает только текстовый список расхождений и статус выгрузки.

## Известные ограничения текущей версии

- Сопоставление слоёв — эвристическое: сначала по имени слоя/id/data-testid/классу, затем по совпадению текста, затем по ближайшему bbox совместимого типа. На неоднозначных именах (`Rectangle 14` и т.п.) возможны неверные пары или отсутствие пары — тогда область попадает в общий пиксельный diff без детальной расшифровки.
- Цвет сравнивается только для сплошных заливок (`fills` с типом SOLID) — градиенты и картиночные заливки не участвуют в семантическом сравнении цвета (но остаются видны в пиксельном diff).
- Режим "APK" в UI присутствует, но пока не работает — сначала будет добавлен статический анализ ресурсов APK, затем автоматизация через ADB/эмулятор.
- Живой скрин веб-страницы работает, только если URL доступен с той же машины, где запущен бэкенд. Если это не так — используйте ручную загрузку скриншота (чекбокс в UI); в этом режиме DOM недоступен, поэтому сравнение будет только пиксельным.
